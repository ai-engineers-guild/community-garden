# Risk Analysis Skill

Этот скилл отвечает за выявление рисков в сообществе. Раньше часть рисков считалась хардкодом, теперь всё анализируется интеллектуально.

## Workflow

1. Прочитай словарь рисков из файла `risk_taxonomy.yml` в папке `.agents/skills/risk_analysis/`.
2. Используя данные из экспорта сообщений, тредов и предыдущих отчетов (из `.garden/gold/reports/`), выяви наличие данных рисков.
3. Не используй медицинские диагнозы (типа "шиза"), используй рабочие поведенческие термины (attention capture, emotional dumping, derailment behavior).
4. Оцени уровень риска и приведи конкретные доказательства (цитаты, паттерны поведения).
5. Результаты сохрани в `.garden/silver/risks/risks_{period}.yml`.

**Важно (Тон и Конкретика):**
- Для каждого риска **обязательно указывай конкретных людей (Имя/Никнейм)**, которые его создают.
- Выступай как объективный аналитик. Если кто-то использует чат для слива эмоций — так и пиши: "Пользователь [Никнейм] использует чат для эмоционального слива". 
- Называй деструктивное поведение деструктивным. Админы должны точно знать, с кем нужно провести беседу.
- Сопоставь с прошлым анализом. Стало ли хуже? Учитывай контекст: не путай здоровую дискуссию с "шумом".
