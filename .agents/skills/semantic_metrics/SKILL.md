# Semantic Metrics Skill

Этот скилл отвечает за вычисление метрик для Health Radar (6 осей).

## Workflow

1. Прочитай реестр метрик из `index.md` и `registry.yml` в папке `.agents/skills/semantic_metrics/`.
2. Обрати внимание на тип метрик (`type: discrete` или `type: semantic`).
3. Для **discrete** метрик, используй предоставленные статистические данные или выполни скрипты на Python (например, вызов функций агрегации из `.garden/silver/metrics/`), чтобы их подсчитать.
4. Для **semantic** метрик, проанализируй логи сообщений и треды, используя LLM (себя), согласно правилу `how_to_calculate` из реестра.
5. Оцени метрики и соотнеси их с предыдущими периодами (прочитай предыдущие отчеты в `.garden/gold/reports/` если они существуют, чтобы понять динамику).
6. Сформируй итоговый балл (0-100) по каждой из 6 осей.
7. Запиши результат в формат YAML в `.garden/silver/findings/semantic_metrics_{period}.yml` и подготовь аналитическую справку.

**Важно (Тон и Конкретика):**
- Сохраняй прошлые анализы. Учитывай их при расчете.
- Для semantic метрик обязательно приводи конкретные доказательства (куски текста).
- Оценивай метрики цинично и объективно, как внешний строгий аудитор. Если ось "Signal vs Noise" пробила дно из-за двух болтунов — прямо укажи их имена/никнеймы. Не прикрывай провалы сообщества вежливыми словами.
